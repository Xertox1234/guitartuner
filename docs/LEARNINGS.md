- 2026-06-18: "Use vDSP everywhere" is subordinate to the byte-identical accuracy proof — vectorizing a *reduction* (`reduce(0,+)` → `vDSP_meanvD`) reorders summation (measured up to ~282 ULP, flips the LS slope in ~19% of cases) and forces a benchmark re-baseline; spend that only where the perf win is real (dominant arithmetic), not on tiny off-hot-path reductions like `PhaseIntegrator.lsSlope`'s means (`k ≤ 140`). Element-wise non-fused ops (`vsadd`/`vsub`) are bit-preserving; FMA (`vsma`) is not, so the proof is path-local (vDSP macOS ≠ scalar Linux). `accuracy.csv` is gitignored → proof is same-platform regenerate-and-diff → [solutions/best-practices/vdsp-subordinate-to-zero-delta-reductions-reorder-2026-06-18.md](solutions/best-practices/vdsp-subordinate-to-zero-delta-reductions-reorder-2026-06-18.md)
- 2026-06-16: `#if os(iOS)` code is type-checked in CI only by the `iOS Simulator` xcodebuild step (not `swift test`, not the macOS build), so an iOS-26-only AVAudioSession symbol (`.allowBluetoothHFP`, the rename of `.allowBluetooth`) compiled on local Xcode 26.5 but red-lit CI on Xcode 16.2 — masked further by incremental local builds and a truncated `--log-failed` that looked like a compiler OOM; use the lowest-SDK spelling (`#available` gates runtime, not symbol existence), prefer a pinned Xcode + required status check over floating `latest-stable` → [solutions/best-practices/ios-only-code-ci-sdk-skew-2026-06-16.md](solutions/best-practices/ios-only-code-ci-sdk-skew-2026-06-16.md)
- 2026-06-16: `assertionFailure` in a catch labeled "non-fatal" crashes Debug builds (SIGTRAP/"signal 5") and no-ops Release — a debug-fatal on a cache write to `Application Support` (not auto-created on iOS) crashed the app + Previews on device only, while Simulator/macOS passed; never assert on recoverable paths, and create the dir before writing → [solutions/runtime-errors/application-support-not-created-ios-debug-assert-crash-2026-06-16.md](solutions/runtime-errors/application-support-not-created-ios-debug-assert-crash-2026-06-16.md)
- 2026-06-15: Metal triple-buffer semaphore has 4 non-obvious constraints: `value: 3`, signal every early-exit, `addCompletedHandler` before `commit()`, capture semaphore by value — any one violation deadlocks the render loop → [solutions/best-practices/metal-triple-buffer-semaphore-2026-06-15.md](solutions/best-practices/metal-triple-buffer-semaphore-2026-06-15.md)
