# TunerEngine — measured accuracy

> **Pending the first CI run.** This file is generated by the headless benchmark
> (`swift run -c release --package-path Packages/TunerEngine Benchmark --out docs/benchmarks`).
> The numbers below are filled from a real **macOS CI** run (the authoring
> environment has no Swift toolchain) — see the *Run accuracy benchmark* step in
> [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml). Until then, treat
> this as a placeholder; the harness, tests, and method are what define the spec.

Run locally on macOS to populate immediately:

```sh
swift run -c release --package-path Packages/TunerEngine Benchmark --compare --out docs/benchmarks
```

The report contains: headline cents error (mean / abs / σ), accuracy by signal
type (pure / harmonic / **inharmonic**) and by range (bass / mid / high), noise
robustness vs SNR, octave-safety on the low strings, the window/hop table, and the
MPM vs YIN vs hybrid comparison that selects the default method.
